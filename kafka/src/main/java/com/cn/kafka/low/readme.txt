
实现使用低级API读取指定topic，指定partition,指定offset的数据。

①找leader
②找offset

1）消费者使用低级API 的主要步骤：
步骤	            主要工作
1	                根据指定的分区从主题元数据中找到主副本     找leader的时候，是从集群中的每一台服务器上面去找
2	                获取分区最新的消费进度
3	                从主副本拉取分区的消息
4	                识别主副本的变化，重试                     找副本的时候，因为leader上有存有副本的信息，所以可以精确的查找
                                                               将这些副本信息都保存下来，那么当leader宕机时，也阔以精确查找


2）方法描述：
findLeader()	    客户端向种子节点发送主题元数据，将副本集加入备用节点
getLastOffset()	    消费者客户端发送偏移量请求，获取分区最近的偏移量
run()	            消费者低级API拉取消息的主要方法
findNewLeader()	    当分区的主副本节点发生故障，客户将要找出新的主副本
